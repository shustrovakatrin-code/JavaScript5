## Лабораторная работа №5 ##
### Работа с DOM-деревом и событиями в JavaScript ###
__Цель работы__
___Цель данной лабораторной работы — изучить основы взаимодействия JavaScript с DOM-деревом на примере веб-приложения для учета личных финансов.___

_В ходе работы было создано приложение, в котором пользователь может добавлять финансовые транзакции, указывать сумму, категорию и описание. После добавления транзакция отображается в таблице, окрашивается в зависимости от суммы, участвует в подсчете общей суммы и может быть удалена._
### Структура проекта ###
_Проект имеет следующую структуру: _

<u> L5/ → index.html, style.css, src/ → index.js, transactions.js, ui.js, utils.js </u>
### Описание файлов проекта ### 
___index.html___ 

Главный файл проекта, содержащий структуру веб-страницы.
 В нём реализована форма для ввода данных транзакции (сумма, категория, описание),
 таблица для отображения списка транзакций, а также блоки для вывода общей суммы и полного описания выбранной транзакции.
    В конце файла подключаются все JavaScript-файлы.
   
___style.css___

Файл стилей, отвечающий за внешний вид страницы. 
 В нём задаются стили для формы, таблицы, кнопок и блоков.
  Также реализовано цветовое выделение транзакций: зеленый цвет для доходов и красный для расходов.

___utils.js___

Содержит вспомогательные функции.
  В частности: генерация уникального идентификатора транзакции,
  форматирование даты и сокращение текста описания до первых четырёх слов. 
  Эти функции используются в других частях программы.

___transactions.js___

Файл отвечает за работу с данными. 
В нём создаётся массив транзакций, а также реализованы функции для добавления транзакции,
удаления, поиска по id и вычисления общей суммы всех транзакций.

___ui.js___

Отвечает за взаимодействие с DOM-деревом. 
 Здесь реализованы функции для отображения транзакций в таблице, 
 удаления строк, обновления общей суммы, вывода полного описания и очистки формы.

___index.js___

Основной файл логики приложения. В нём происходит получение элементов страницы, 
обработка событий (отправка формы, клики по таблице),
 создание новых транзакций и вызов функций из других файлов.

 ### Назначение разделения кода ###

Код проекта разделён на несколько файлов для того, чтобы каждая часть программы отвечала за свою задачу.

`utils.js` содержит вспомогательные функции.  
`transactions.js` отвечает за хранение и обработку данных.  
`ui.js` отвечает за отображение данных на странице.  
`index.js` связывает все части программы и обрабатывает действия пользователя.

Такое разделение делает код более понятным, удобным для чтения и дальнейшего изменения.

 ### Работа с данными ###

___Массив транзакций___

Все транзакции хранятся в массиве `transactions`.

Каждый элемент массива — это объект со следующими полями:
- id — уникальный идентификатор
- date — дата и время
- amount — сумма
- category — категория
- description — описание

С массивом выполняются основные операции:
- добавление транзакции
- удаление транзакции
- поиск по id
- подсчет общей суммы

## Реализация функционала ##

___Добавление транзакции___

При вводе данных в форму и нажатии кнопки "Добавить" происходит обработка события submit.

 Сначала выполняется проверка на заполненность всех полей и корректность введённой суммы. Затем в зависимости от категории меняется знак значения суммы(если зарплата, то остается положительной), затем создаётся объект транзакции, который добавляется в массив и отображается в таблице.

___Валидация данных___

Перед добавлением транзакции выполняется проверка введённых данных.

Проверяется:
- заполнены ли все поля формы
- является ли сумма числом

Если данные введены некорректно, пользователю выводится сообщение об ошибке, и транзакция не добавляется.

Это позволяет избежать появления некорректных данных в таблице.

___Отображение транзакций___

Каждая транзакция отображается в виде строки таблицы.

 Дата форматируется, описание сокращается до четырёх слов.
 
  В зависимости от значения суммы строка получает соответствующий цвет.

___Удаление транзакции___

Для каждой строки таблицы создаётся кнопка удаления.

 При нажатии на неё транзакция удаляется из массива и из таблицы.
 
  Для обработки используется делегирование событий — обработчик назначается на всю таблицу.

___Подсчет общей суммы___

После каждого добавления или удаления транзакции вызывается функция подсчета общей суммы.

 Полученное значение выводится на страницу.

___Отображение полного описания___

При нажатии на строку таблицы отображается полное описание выбранной транзакции в отдельном блоке под таблицей.

### Работа с событиями ###

В приложении используются события JavaScript для взаимодействия с пользователем.

___Событие submit___

Используется для обработки отправки формы.

Перед выполнением используется метод:

> event.preventDefault()

Он отменяет стандартное поведение браузера (перезагрузку страницы).

___Событие click___

Используется для обработки нажатий:
- на кнопку удаления
- на строку таблицы

___Всплытие событий___

События в DOM распространяются вверх по дереву (всплытие).

Это используется для делегирования событий, когда обработчик назначается на родительский элемент.

___Использование event.target и closest___

В обработчике клика используется `event.target`.

`event.target` показывает, на какой именно элемент нажал пользователь.

Также используется метод `closest("tr")`, который помогает найти ближайшую строку таблицы, даже если клик был сделан по ячейке внутри строки.

Это нужно для того, чтобы при нажатии на строку таблицы можно было получить нужную транзакцию и вывести её полное описание.

## Документирование кода ##

__документирования функции__
```
/**
 * Генерирует уникальный id для транзакции.
 * @returns {string} Уникальный идентификатор.
 */
function generateId() {
  return Date.now().toString();
} 

```
Данная функция не принимает параметров и возвращает строку.

Она используется для создания уникального идентификатора каждой транзакци

__Пример функции с параметром__
```
/**
 * Форматирует дату.
 * @param {string} dateString - Дата в виде строки.
 * @returns {string} Отформатированная дата.
 */
function formatDate(dateString) {
  const date = new Date(dateString);
  return date.toLocaleString("ru-RU");
} 
```
Здесь уже используется параметр dateString.

__функции обработки данных__
```  
/**
 * Возвращает первые 4 слова описания.
 * @param {string} text - Полное описание.
 * @returns {string} Краткое описание.
 */
function getShortDescription(text) {
  return text.split(" ").slice(0, 4).join(" ");
}
```
Эта функция используется для отображения краткого описания транзакции.

## Контрольные вопросы ##

### ___1. Каким образом можно получить доступ к элементу на веб-странице с помощью JavaScript?___

_Доступ к элементам осуществляется с помощью методов DOM._
``` 
const form = document.querySelector("#transaction-form");
const amountInput = document.querySelector("#amount");
const table = document.querySelector("#transactions-table");
``` 
Метод querySelector() позволяет получить элемент по CSS-селектору.

### ___2. Что такое делегирование событий и как оно используется?___

_Делегирование событий — это способ обработки событий через родительский элемент._

В программе обработчик клика назначен на таблицу:
```
table.addEventListener("click", function (event) {
```
При клике используется:
>event.target

Он показывает, на какой элемент нажал пользователь.

Если нажата кнопка удаления:
>if (event.target.className === "delete-btn")

Тогда транзакция удаляется из массива и из таблицы.

___Таким образом, один обработчик управляет всеми кнопками.___ 

### ___3. Как можно изменить содержимое элемента DOM после его выборки?___

Содержимое элемента можно изменить с помощью:

1. textContent — изменяет текст
2. innerHTML — позволяет вставлять HTML

_Пример:_
>totalAmount.textContent = calculateTotal().toFixed(2);

Обновляется общая сумма.
>fullDescription.textContent = text;

Отображается полное описание.
```
row.innerHTML = `
  <td>...</td>
`;
```
Заполняется строка таблицы.
### ___4. Как можно добавить новый элемент в DOM дерево с помощью JavaScript?___

Для добавления элемента используются методы:

>const row = document.createElement("tr");

Создаётся новый элемент.

>tableBody.appendChild(row);

Элемент добавляется в таблицу.

Перед этим строка заполняется данными:
```
row.innerHTML = `
  <td>${formatDate(transaction.date)}</td>
  <td>${transaction.category}</td>
  <td>${getShortDescription(transaction.description)}</td>
  <td><button class="delete-btn">Удалить</button></td>
`;
```

_Таким образом, новые элементы появляются на странице без её перезагрузки._
